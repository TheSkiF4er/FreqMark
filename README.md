# Частотные отметки

Готовый статический сайт для вашей таблицы частот.

## Что уже сделано

- пароль для входа на страницу
- отметка только числовых ячеек
- выбор любого цвета
- легенда `цвет -> текст`
- синхронизация отметок и легенды между устройствами через Supabase
- мобильная версия:
  - на широком экране обычная таблица
  - на телефоне карточки по группам с крупными кнопками

## Что лежит в архиве

- `index.html` — основная страница
- `styles.css` — стили
- `app.js` — логика сайта
- `config.js` — сюда вставить URL и Anon Key от Supabase
- `supabase-schema.sql` — SQL для создания таблиц и Realtime

## Быстрый запуск

### 1) Создайте проект в Supabase
Зайдите в Supabase и создайте новый проект.

### 2) Выполните SQL
Откройте SQL Editor и вставьте содержимое файла `supabase-schema.sql`.

По умолчанию пароль страницы уже задан:
- пароль: `fpv58`

### 3) Заполните `config.js`
Откройте `config.js` и вставьте туда:
- `supabaseUrl`
- `supabaseAnonKey`

### 4) Запустите сайт
Это статический сайт, поэтому достаточно открыть его через любой простой сервер.

Примеры:

#### Вариант A — Python
```bash
python3 -m http.server 8080
```

#### Вариант B — VS Code Live Server
Откройте папку и запустите Live Server.

#### Вариант C — GitHub Pages / Netlify / Vercel
Можно загрузить эти файлы как обычный статический сайт.

После запуска откройте:
- `http://localhost:8080`

## Как работает синхронизация
- отметки ячеек хранятся в таблице `cell_marks`
- подписи цветов хранятся в таблице `legend_items`
- изменения приходят на другие устройства через Supabase Realtime

## Подсказка
Если после запуска сайт пишет, что нужна настройка:
- проверьте `config.js`
- убедитесь, что SQL из `supabase-schema.sql` выполнен полностью
- убедитесь, что таблицы `cell_marks` и `legend_items` добавлены в Realtime
